03 CDC Engine ‚Äî Polars In-Memory Comparison
üîç
Hash-Based Change Detection
Compares fresh extraction (df_fresh) against existing Stage current rows (df_existing) using Polars joins on primary key columns. PK dtype alignment (P0-12) runs first to prevent silent join failures. Count invariant validated: inserts + updates + unchanged == total_fresh.

Classification via Polars Joins

INSERT ‚Äî Anti-Join
Rows in df_fresh but NOT in df_existing. PK has never been seen. New rows in the source system.
UPDATE ‚Äî Inner Join + Hash ‚â†
PK exists in both, but _row_hash differs. Source data changed. NULL hash guard (P0-10) treats missing hashes as "changed".
DELETE ‚Äî Reverse Anti-Join
Rows in df_existing but NOT in df_fresh. Row removed from source. Windowed: scoped to extraction window only (P1-4).
UNCHANGED ‚Äî Inner Join + Hash =
PK exists in both and hashes match exactly. No change detected. Row is carried forward as-is.
‚ö° INSERT-First Crash Safety (P0-9)
New CDC rows are INSERTed before old rows are expired. A crash after INSERT but before EXPIRE leaves recoverable duplicate current rows ‚Äî not zero current rows (data loss). The next run's dedup step (L-1) corrects automatically.

UDM_BRONZE ‚Äî Slowly Changing Dimension Type 2
05 SCD2 Engine ‚Äî Bronze Version Management
üèõÔ∏è
SCD2 Comparison ‚Äî _row_hash vs UdmHash
Compares CDC current rows against active Bronze rows (UdmActiveFlag=1). Anti-join detects new inserts. Inner join + hash comparison detects new versions vs unchanged. Resurrections (E-18) tracked when deleted PKs reappear. Count invariant: inserts + new_versions + unchanged == CDC unique PKs.
INSERT New Versions
New rows inserted with UdmActiveFlag=1. Updates & resurrections inserted with UdmActiveFlag=0 (inactive ‚Äî waiting for activation after old versions close).
CLOSE Old Versions
Old active versions set to UdmActiveFlag=0, UdmEndDateTime=NOW(). Batched at 4,000 rows (B-2) to prevent lock escalation. Via staging table + UPDATE JOIN.
ACTIVATE New Versions
New versions from Step 1 promoted to UdmActiveFlag=1 via _activate_new_versions(). After this, exactly one active version exists per PK.
üîí P0-8 ‚Äî INSERT-First Crash Safety
Same principle as CDC: INSERT first, then UPDATE. A crash after Step 1 leaves orphaned Flag=0 rows (cleaned up by _cleanup_orphaned_inactive_rows() on next run). A crash after reverse order would leave zero active rows ‚Äî unrecoverable data loss.
‚õî Bronze is APPEND-ONLY
SCD2 only closes old versions and inserts new ones. Bronze tables must NEVER be truncated. Deleting from Bronze destroys the audit trail permanently. If rows need correction, insert a new version ‚Äî never modify history.
